# Rubric: startup_naming

**Domain:** creative_naming
**Total Points:** 34
**Pass Threshold:** 0.75

## Criterion 1: name_count
**Category:** structure
**Max Points:** N/A
**Description:** Exactly 5 names provided, each with rationale
**Pass Condition:** Exactly 5 names. Each has 1-2 sentence rationale explaining the thinking behind it.

## Criterion 2: name_memorability
**Category:** quality
**Max Points:** N/A
**Description:** Names are memorable and pronounceable
**Pass Condition:** Each name: ≤2 syllables, contains no more than 1 consonant in a row, uses only the most common English phonetic patterns (CVC, CVCV, VCVC), and could be easily spelled after hearing it once.

## Criterion 3: name_domain_resonance
**Category:** relevance
**Max Points:** N/A
**Description:** Names clearly signal legal/contract/AI domain through specific word choices
**Pass Condition:** Each name contains at least two elements from different categories: legal terminology (contract, brief, clause, legal, law, etc.), trust/precision concepts (precise, secure, vault, sharp, etc.), or AI/technology (intel, smart, auto, neural, etc.). Names must avoid generic AI prefixes/suffixes and create meaningful semantic connections between domains.

## Criterion 4: name_availability
**Category:** practicality
**Max Points:** N/A
**Description:** Names have structural characteristics that suggest domain availability
**Pass Condition:** At least 3/5 names are: not common English dictionary words, not obvious combinations of common words, and include made-up or coined elements. Response includes brief rationale for why each name might be available.

## Criterion 5: name_variety
**Category:** range
**Max Points:** N/A
**Description:** Names span different naming strategies — not all the same pattern
**Pass Condition:** Mix of approaches: metaphorical, coined, real-word-repurposed, compound, classical/Latin root. At least 3 different strategies.
